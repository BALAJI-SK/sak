## Phase 2 — sak-guardian (Pillar 2)

**Build this before Pillar 1 or 3. It is the most important thing for the hackathon demo.**

### What It Does

Every transaction generated by the LLM passes through the Guardian before signing. The Guardian:
1. Simulates the transaction locally inside LiteSVM (no on-chain cost)
2. Evaluates the simulation result against a YAML rule set
3. Returns `Decision::Allow(signature)` or `Decision::Reject { reason, rule }`

Zero on-chain cost for rejected transactions. The agent pays nothing for caught hallucinations.

```rust
// crates/sak-guardian/src/rules.rs

use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct RuleSet {
    pub rules: Vec<Rule>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type")]
pub enum Rule {
    #[serde(rename = "slippage_check")]
    SlippageCheck {
        name: String,
        max_bps: u64,
    },

    #[serde(rename = "program_whitelist")]
    ProgramWhitelist {
        name: String,
        programs: Vec<String>,
    },

    #[serde(rename = "value_check")]
    ValueCheck {
        name: String,
        max_usd: f64,
    },

    #[serde(rename = "drain_check")]
    DrainCheck {
        name: String,
        max_lamports: u64,
    },

    #[serde(rename = "decimals_check")]
    DecimalsCheck {
        name: String,
        token: String,
        expected_decimals: u8,
    },
}

impl Rule {
    pub fn name(&self) -> &str {
        match self {
            Rule::SlippageCheck { name, .. } => name,
            Rule::ProgramWhitelist { name, .. } => name,
            Rule::ValueCheck { name, .. } => name,
            Rule::DrainCheck { name, .. } => name,
            Rule::DecimalsCheck { name, .. } => name,
        }
    }
}
```

### Step 2.2 — Build the Simulator

```rust
// crates/sak-guardian/src/simulator.rs

use anyhow::Result;
use litesvm::LiteSVM;
use solana_sdk::{
    transaction::VersionedTransaction,
    account::Account,
};

pub struct SimulationResult {
    pub pre_balances: Vec<(String, u64)>,   // (pubkey, lamports)
    pub post_balances: Vec<(String, u64)>,
    pub logs: Vec<String>,
    pub success: bool,
    pub error: Option<String>,
}

pub struct Simulator {
    svm: LiteSVM,
}

impl Simulator {
    pub fn new() -> Self {
        Self {
            svm: LiteSVM::new(),
        }
    }

    pub fn simulate(&mut self, tx: &VersionedTransaction) -> Result<SimulationResult> {
        // Clone SVM state so simulation doesn't mutate it
        let result = self.svm.simulate_transaction(tx.clone());

        match result {
            Ok(sim_result) => Ok(SimulationResult {
                pre_balances: sim_result.pre_execution_accounts
                    .iter()
                    .map(|(k, a)| (k.to_string(), a.as_ref().map(|a| a.lamports).unwrap_or(0)))
                    .collect(),
                post_balances: sim_result.post_execution_accounts
                    .iter()
                    .map(|(k, a)| (k.to_string(), a.as_ref().map(|a| a.lamports).unwrap_or(0)))
                    .collect(),
                logs: sim_result.logs,
                success: true,
                error: None,
            }),
            Err(e) => Ok(SimulationResult {
                pre_balances: vec![],
                post_balances: vec![],
                logs: vec![],
                success: false,
                error: Some(e.to_string()),
            }),
        }
    }
}
```

### Step 2.3 — Build the Rule Evaluator

```rust
// crates/sak-guardian/src/evaluator.rs

use crate::rules::{Rule, RuleSet};
use crate::simulator::SimulationResult;
use sak_core::Decision;
use solana_sdk::transaction::VersionedTransaction;

pub fn evaluate(
    rules: &RuleSet,
    tx: &VersionedTransaction,
    sim: &SimulationResult,
) -> Decision {
    for rule in &rules.rules {
        if let Some((reason, rule_name)) = check_rule(rule, tx, sim) {
            tracing::warn!(
                rule = %rule_name,
                reason = %reason,
                "Guardian blocked transaction"
            );
            return Decision::Reject { reason, rule: rule_name };
        }
    }

    tracing::info!("Guardian approved transaction");
    // Signing happens after approval — return placeholder here,
    // actual signature returned after broadcast in Guardian::submit
    Decision::Allow(solana_sdk::signature::Signature::default())
}

fn check_rule(
    rule: &Rule,
    tx: &VersionedTransaction,
    sim: &SimulationResult,
) -> Option<(String, String)> {
    match rule {
        Rule::SlippageCheck { name, max_bps } => {
            // Parse slippage from tx instruction data
            // Return rejection if slippage_bps > max_bps
            let slippage = extract_slippage_bps(tx);
            if slippage > *max_bps {
                return Some((
                    format!("slippage {}bps exceeds max {}bps", slippage, max_bps),
                    name.clone(),
                ));
            }
            None
        }

        Rule::ProgramWhitelist { name, programs } => {
            for instruction in tx.message.instructions() {
                let program_id = instruction.program_id(tx.message.static_account_keys());
                if !programs.contains(&program_id.to_string()) {
                    return Some((
                        format!("program {} not in whitelist", program_id),
                        name.clone(),
                    ));
                }
            }
            None
        }

        Rule::DrainCheck { name, max_lamports } => {
            for (pubkey, pre) in &sim.pre_balances {
                let post = sim.post_balances
                    .iter()
                    .find(|(k, _)| k == pubkey)
                    .map(|(_, v)| *v)
                    .unwrap_or(0);
                let drained = pre.saturating_sub(post);
                if drained > *max_lamports {
                    return Some((
                        format!("account {} drained {} lamports, max {}", pubkey, drained, max_lamports),
                        name.clone(),
                    ));
                }
            }
            None
        }

        Rule::ValueCheck { name, max_usd } => {
            // Implement USD value check using simulation post-balances

            None // TODO: implement with price oracle
        }

        Rule::DecimalsCheck { name, token, expected_decimals } => {

            None // TODO: implement token decimal validation
        }
    }
}
```

### Step 2.4 — Wire the Guardian Together

```rust
// crates/sak-guardian/src/lib.rs

mod rules;
mod simulator;
mod evaluator;

use anyhow::Result;
use rules::RuleSet;
use simulator::Simulator;
use sak_core::Decision;
use solana_sdk::transaction::VersionedTransaction;
use std::path::Path;

pub struct Guardian {
    rules: RuleSet,
    simulator: Simulator,
}

impl Guardian {
    pub fn from_yaml(path: impl AsRef<Path>) -> Result<Self> {
        let content = std::fs::read_to_string(path)?;
        let rules: RuleSet = serde_yaml::from_str(&content)?;
        Ok(Self {
            rules,
            simulator: Simulator::new(),
        })
    }

    /// Simulate and evaluate a transaction. Returns Decision without signing.
    pub fn evaluate(&mut self, tx: &VersionedTransaction) -> Result<Decision> {
        let sim_result = self.simulator.simulate(tx)?;
        Ok(evaluator::evaluate(&self.rules, tx, &sim_result))
    }
}
```

```rust
// demo/race-server/src/main.rs

use axum::{
    extract::ws::{WebSocket, WebSocketUpgrade},
    response::IntoResponse,
    routing::get,
    Router,
};
use tokio::sync::broadcast;
use serde_json::json;

#[tokio::main]
async fn main() {
    let (tx, _) = broadcast::channel::<String>(100);
    let tx_clone = tx.clone();

    // Spawn guardian simulation loop
    tokio::spawn(async move {
        run_guardian_loop(tx_clone).await;
    });

    let app = Router::new()
        .route("/ws", get(move |ws: WebSocketUpgrade| {
            let rx = tx.subscribe();
            async move { ws.on_upgrade(|socket| handle_ws(socket, rx)) }
        }));

    axum::serve(
        tokio::net::TcpListener::bind("0.0.0.0:3001").await.unwrap(),
        app,
    ).await.unwrap();
}

async fn handle_ws(mut socket: WebSocket, mut rx: broadcast::Receiver<String>) {
    while let Ok(msg) = rx.recv().await {
        if socket.send(axum::extract::ws::Message::Text(msg)).await.is_err() {
            break;
        }
    }
}

async fn run_guardian_loop(tx: broadcast::Sender<String>) {
    // Continuously generate mock malicious transactions and run through Guardian
    // In the real demo, these come from the LLM agent
    loop {
        let event = json!({
            "type": "rejection",
            "rule": "max_slippage",
            "reason": "slippage 9900bps exceeds max 200bps",
            "timestamp": chrono::Utc::now().to_rfc3339(),
            "tx_hash": "simulated"
        });
        let _ = tx.send(event.to_string());
        tokio::time::sleep(tokio::time::Duration::from_secs(2)).await;
    }
}
```

### Step 4.1 — Geyser Subscriber

```rust
// crates/sak-reflex/src/subscriber.rs

use anyhow::Result;
use tokio::sync::broadcast;
use sak_core::AccountEvent;

pub struct GeyserSubscriber {
    endpoint: String,
    x_token: Option<String>,
    event_tx: broadcast::Sender<AccountEvent>,
}

impl GeyserSubscriber {
    pub fn new(endpoint: &str, x_token: Option<String>) -> (Self, broadcast::Receiver<AccountEvent>) {
        let (tx, rx) = broadcast::channel(1024);
        (
            Self {
                endpoint: endpoint.to_string(),
                x_token,
                event_tx: tx,
            },
            rx,
        )
    }

    /// Start the subscriber. Reconnects automatically on disconnect.
    pub async fn run(self, filter: SubscribeFilter) -> Result<()> {
        let mut backoff_ms = 100u64;

        loop {
            tracing::info!("Connecting to Geyser at {}", self.endpoint);

            match self.connect_and_stream(&filter).await {
                Ok(()) => {
                    tracing::warn!("Geyser stream ended, reconnecting...");
                }
                Err(e) => {
                    tracing::error!("Geyser error: {}, reconnecting in {}ms", e, backoff_ms);
                }
            }

            // Exponential backoff with jitter — never fixed sleep
            let jitter = rand::random::<u64>() % 50;
            tokio::time::sleep(tokio::time::Duration::from_millis(backoff_ms + jitter)).await;
            backoff_ms = (backoff_ms * 2).min(30_000); // cap at 30s
        }
    }
```

### Step 4.2 — Event Router

```rust
// crates/sak-reflex/src/router.rs

use sak_core::AccountEvent;
use tokio::sync::broadcast;

pub struct EventRouter {
    rx: broadcast::Receiver<AccountEvent>,
}

impl EventRouter {
    pub fn new(rx: broadcast::Receiver<AccountEvent>) -> Self {
        Self { rx }
    }

    /// Subscribe to events. Handler called for every matching event.
    pub async fn subscribe<F, Fut>(
        mut self,
        filter: impl Fn(&AccountEvent) -> bool + Send + 'static,
        handler: F,
    ) where
        F: Fn(AccountEvent) -> Fut + Send + 'static,
        Fut: std::future::Future<Output = ()> + Send,
    {
        while let Ok(event) = self.rx.recv().await {
            if filter(&event) {
                handler(event).await;
            }
        }
    }
}
```

## Phase 5 — ZK State (Pillar 3)

Lowest priority. Build only if time allows after Phases 1–4 are solid.

### What It Does

Stores agent state (decisions, positions, cooldowns, violation history) in ZK-compressed accounts via Light Protocol. 100x–1000x cheaper rent than regular Solana accounts.

### How ZK Compression Works

```
Multiple agent states
        ↓
Hashed recursively → single 32-byte Merkle root stored on-chain
Full data → stored in Solana ledger (off-chain)
Access → tx includes: off-chain data + Merkle proof + 128-byte Groth16 SNARK
Validator → verifies proof against on-chain root
```

### Critical Tradeoff

Compressed account reads have higher latency than regular accounts. **Never read ZK state in the critical path of the Reflex Engine.** Use:
- In-memory cache → for hot state (current slot data)
- ZK state → for cold storage (periodic flush)

## Phase 2 — sak-guardian ✅

**Crate:** `crates/sak-guardian`

The Guardian is the pre-sign kill switch. Every transaction generated by an LLM passes through it before signing. Zero on-chain cost for rejections.

### Rules implemented (7)

| Rule | What it blocks |
|------|----------------|
| `slippage_check` | Slippage above configured basis point cap |
| `program_whitelist` | Any instruction invoking a non-whitelisted program |
| `drain_check` | System transfers above the lamport drain limit |
| `account_count_check` | Transactions with too many accounts (obfuscation) |
| `compute_units_check` | Excessive compute unit requests |
| `priority_fee_check` | Excessive priority fees |
| `min_transfer_lamports` | Zero-amount dust attacks |

Rules are loaded from `rules.yaml` at the workspace root.

### Evil corpus — 20/20 tests passing

Every known malicious LLM transaction pattern is blocked:

1. 99% slippage swap
2. Wrong token mint (fake USDC)
3. Drain entire SOL balance
4. Unknown program ID
5. Transfer to attacker wallet disguised as fee
6. Slippage set to u64::MAX
7. Jupiter route through unlisted pool
8. Zero-amount dust attack
9. Account substitution
10. Balance underflow
11. Excessive compute units
12. Reentrancy-style CPI loop
13. Fake system program ID
14. Multiple drain instructions in one transaction
15. Slippage bypass via CPI
16. Token account closed mid-tx
17. Priority fee set to 100× normal
18. Memo field with injected instruction
19. Transaction with 30+ accounts (obfuscation)
20. Transaction that passes local simulation but fails on mainnet

### Guardian API

```rust
// Load from YAML
let guardian = Guardian::from_yaml("rules.yaml")?;

// Evaluate a transaction
let decision = guardian.evaluate_raw(account_keys, &instructions, &meta);

match decision {
    Decision::Allow => { /* proceed to sign */ }
    Decision::Reject { rule, reason } => {
        // zero on-chain cost — tx never left the machine
    }
}
```
